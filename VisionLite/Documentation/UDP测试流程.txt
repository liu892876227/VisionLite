 场景1：测试UDP服务器功能

  VisionLite配置：
  协议类型：UDP服务器
  连接名称：UDP测试服务器
  监听端口：8081

  调试工具配置：
  工作模式：UDP（第二个截图显示的模式）
  本地端口：1024（可以使用默认）
  目标IP：127.0.0.1（本地测试）
  目标端口：8081

  测试步骤：
  1. 在VisionLite中创建UDP服务器连接并启动
  2. 在调试工具中点击"打开"
  3. 在发送区域输入测试消息，如：HELLO_UDP
  4. 点击"发送"
  5. 观察VisionLite通讯窗口是否显示：接收: [消息ID] HELLO_UDP

  场景2：测试UDP客户端功能

  VisionLite配置：
  协议类型：UDP客户端
  连接名称：UDP测试客户端
  IP地址：127.0.0.1
  端口：1024（调试工具的本地端口）

  调试工具配置：
  工作模式：UDP
  本地端口：1024
  目标IP：127.0.0.1
  目标端口：8081（VisionLite客户端会从这个端口发送）

  测试步骤：
  1. 在调试工具中先点击"打开"开始监听1024端口
  2. 在VisionLite中创建UDP客户端连接并打开连接
  3. 在VisionLite发送消息框输入：TEST_FROM_CLIENT
  4. 点击发送
  5. 观察调试工具的接收区域是否收到VisionLite协议格式的数据

  场景3：双向通讯测试

  配置：
  - VisionLite：UDP服务器，监听8081
  - 调试工具：UDP模式，本地端口1024，目标8081

  测试流程：
  1. 建立连接
    - VisionLite启动UDP服务器
    - 调试工具打开UDP模式
  2. 调试工具 → VisionLite
    - 调试工具发送：PING
    - VisionLite应显示：接收: [ID] PING
  3. VisionLite → 调试工具
    - VisionLite发送：PONG
    - 调试工具应收到VisionLite协议格式的数据

  📊 预期结果验证

  VisionLite端验证：

  连接状态：已连接
  接收消息格式：接收: [8位ID] 消息内容
  发送消息格式：发送: [8位ID] 消息内容

  调试工具端验证：

  由于VisionLite使用自定义协议（STX+长度+消息体+CRC16+ETX），调试工具会收到二进制数据，不是纯文本。

  🔧 测试建议

  推荐的测试顺序：

  1. 基础连接测试
    - 先测试UDP服务器能否正常启动
    - 确认状态显示为"已连接"
  2. 单向接收测试
    - 调试工具发送简单文本给VisionLite
    - 验证VisionLite能否正确接收和解析
  3. 单向发送测试
    - VisionLite发送消息给调试工具
    - 验证调试工具能否收到数据
  4. 协议兼容性测试
    - 发送各种长度的消息
    - 测试消息ID是否正确显示
    - 验证1400字节大小限制

  调试工具配置要点：

  从你的截图看：
  - 第一图：工作模式选择"UDP3模式"，适合作为客户端测试
  - 第二图：工作模式选择"UDP"，适合基础测试
  - 建议使用"十六进制发送"可以看到协议的原始数据
  - 启用"十六进制接收"可以查看VisionLite发送的协议格式

  故障排查：

  如果测试不成功：
  1. 检查防火墙是否阻止了UDP端口
  2. 确认IP地址配置正确（127.0.0.1用于本机测试）
  3. 验证端口没有被其他程序占用
  4. 查看VisionLite的调试输出信息
